title: Server Platforms

`MauriceNino/minecraft-server` manages the downloading, linking, and execution of your core server JAR. It takes the heavy lifting out of managing Java processes and garbage collection.

## Supported Platforms

You can control which software runs via the `TYPE` environment variable.

### Game Servers

- **`PAPER`** (Default): High-performance Spigot fork. Recommended for most plugin-based servers.
- **`FOLIA`**: Regionized multithreading Paper fork for massive player counts.
- **`VANILLA`**: The official server software from Mojang. Reliable and pure, but lacks plugin support.
- **`SPIGOT`**: The most popular plugin-compatible fork of CraftBukkit.
- **`BUKKIT`**: The classic CraftBukkit server software.

### Proxies

- **`VELOCITY`**: The next-generation proxy for maximum performance and stability.
- **`WATERFALL`**: A high-performance fork of BungeeCord, compatible with Bungee plugins.


## Version Resolution

The orchestrator dynamically fetches builds from various official and community APIs:
- **PaperMC API:** For `PAPER`, `FOLIA`, `VELOCITY`, and `WATERFALL`.
- **Mojang API:** For `VANILLA`.
- **GetBukkit API:** For `SPIGOT` and `BUKKIT`.

### `VERSION`

Specifies the game or proxy version.

- **Default:** `latest`
- **Aliases:** `latest` or `stable` resolves to the most recent stable release. `experimental` or `beta` resolves to the latest experimental build (where supported).
- **Exact:** e.g., `1.21.4`

### `BUILD`

Specifies the exact build number. Only applicable to **PaperMC-based platforms**. For other types, this is usually ignored and defaults to `latest`.

- **Default:** `latest`
- **Aliases:** `latest` resolves to the very highest build number available for the selected version.
- **Exact:** e.g., `123`

## Java Execution Context

When executing the server JAR, the orchestrator also manages the JVM environment.

### `MEMORY`

Defines the heap size. It is passed exactly as written to both `-Xms` and `-Xmx`.

- **Default:** `1G` (e.g., `2G`, `512M`)

### `JVM_FLAGS`

Space-separated list of flags passed to the `java` execution command.

- **Default:** None (Empty). However, if left empty, the orchestrator **injects highly optimized defaults**:
  - For **Game Servers** (`PAPER`, `FOLIA`, `VANILLA`, `SPIGOT`, `BUKKIT`): Injects Aikar's flags (`-Dusing.aikars.flags`, `-XX:+UseG1GC`, G1GC tuning parameters, etc.) along with `--add-modules=jdk.incubator.vector`.
  - For **Proxies** (`VELOCITY`, `WATERFALL`): Injects proxy-specific optimized G1GC flags (`-XX:G1HeapRegionSize=4M`, `-XX:MaxInlineLevel=15`, etc.).

If you provide _any_ flags to `JVM_FLAGS`, these defaults are skipped completely, allowing you full control.
